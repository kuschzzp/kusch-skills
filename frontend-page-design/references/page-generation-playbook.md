# 从0生成页面流程

目标：在满足用户风格偏好的前提下，从空白状态产出结构清晰、视觉统一、可响应式适配的页面。

## 1) 需求澄清（先问再做）

至少确认以下信息：
- 页面类型：官网首页 / 落地页 / 仪表盘 / 个人主页 / 文档页 / 电商页
- 核心目标：品牌展示 / 线索收集 / 转化购买 / 信息阅读 / 数据查看
- 目标用户：B 端 / C 端 / 开发者 / 内部团队
- 必要区块：Hero、特性区、对比区、案例区、FAQ、页脚等
- 技术栈与限制：原生 HTML/CSS、React、Vue、Tailwind、是否可用第三方库

## 2) 信息架构优先

- 先输出页面区块顺序，再写样式。
- 每个区块只承担一个主要目的。
- 明确主 CTA 和次 CTA，避免多个同权重主操作。
- 内容密度从上到下递进，首屏聚焦核心价值。

## 3) 布局骨架优先

- 先建立容器宽度、网格、断点和基础间距系统。
- 先保证移动端可读，再增强桌面端布局表现。
- 避免在结构未稳定前进行高成本视觉细节雕琢。

## 4) 视觉系统落地

- 根据用户确认的色板建立颜色 token。
- 建立字体层级（标题、正文、说明、按钮标签）。
- 建立组件规范（按钮、卡片、表单、导航、状态样式）。
- 动效与毛玻璃按用户确认策略执行，不擅自添加。

## 5) 质量门槛

- 响应式：移动端/平板/桌面端无明显断裂。
- 可访问性：语义化标签、焦点可见、对比度可读。
- 一致性：间距、字号、圆角、阴影、边框遵循统一 token。
- 可维护性：避免堆叠魔法值，优先变量与复用样式。

## 输出建议

对用户汇报时，按以下顺序：
1. 页面结构方案（区块与目标）
2. 视觉风格确认（色板/动效/毛玻璃等）
3. 已实现内容（布局、组件、交互）
4. 校验结果（响应式、可访问性、功能）
